# Linux Security Log Analysis Using Splunk

## Project Overview

This project demonstrates how Splunk can be used to investigate Linux security events and identify suspicious network activity. Using Splunk Search Processing Language (SPL), I analyzed log data, identified the top source IP address, and documented the investigation process.

## Skills Demonstrated

- Splunk Enterprise
- SPL (Search Processing Language)
- Log Analysis
- Security Event Investigation
- Network Traffic Analysis
- Dashboard Analysis
- Cybersecurity Documentation

## Investigation Summary

The investigation focused on identifying the most active source IP address observed in the log data.

Findings showed that IP address **211.107.232.1** generated 22 FTP-related events, making it the most active source in the dataset.
